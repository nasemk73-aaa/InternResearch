## Task
Review and improve passed prompts and skills

1. Rewrite for clarity
   - Remove ambiguity
   - Use direct instructions
   - Make behavior reliably executable

2. Add structure (if missing)
   - Purpose
   - When to use
   - When NOT to use
   - Step-by-step behavior
   - Output format

3. Add examples
   - Positive examples (correct usage)
   - Negative examples (incorrect usage)
   - Use realistic, concrete cases

4. Preserve intent
   - Do NOT change the skill’s core behavior
   - Only clarify and improve reliability
   - Do not add opinions instead of rules
   - Avoid superficial wording edits without improving execution

## Writing Rules
- Use imperative language ("Do X")
- Avoid vague words (properly, nicely, appropriately, etc.)
- Prefer measurable criteria
- Keep formatting consistent across skills

# Generate CRUD for Module

Generate a complete CRUD scaffold inside a LaraDashboard module using `module:make-crud`.

## Instructions

1. Parse the arguments: $ARGUMENTS
   - Expected format: `ModuleName ModelName field1:type,field2:type,...`
   - Or: `ModuleName --migration=migration_name`
   - If arguments are missing, ask the user interactively

2. Build the artisan command:
   ```
   php artisan module:make-crud {ModuleName} --model={ModelName} --fields="{fields}" --no-interaction
   ```

3. Run the command and capture output

4. After generation, verify key files exist:
   - Model: `modules/{module}/app/Models/{Model}.php`
   - Controller: `modules/{module}/app/Http/Controllers/{Model}Controller.php`
   - Service: `modules/{module}/app/Services/{Model}Service.php`
   - FormRequest: `modules/{module}/app/Http/Requests/{Model}Request.php`
   - Views: `modules/{module}/resources/views/pages/{models}/`
   - Policy: `modules/{module}/app/Policies/{Model}Policy.php`

5. **Post-generation enhancements** (not generated by the command):

### Add Hook Enums
Create `app/Enums/Hooks/{Model}ActionHook.php`:
```php
enum {Model}ActionHook: string
{
    case CREATED_BEFORE = 'action.{model_snake}.created_before';
    case CREATED_AFTER = 'action.{model_snake}.created_after';
    case UPDATED_BEFORE = 'action.{model_snake}.updated_before';
    case UPDATED_AFTER = 'action.{model_snake}.updated_after';
    case DELETED_BEFORE = 'action.{model_snake}.deleted_before';
    case DELETED_AFTER = 'action.{model_snake}.deleted_after';
}
```

### Wire Hooks into Service
Add `Hook::doAction()` calls in the generated service's create/update/delete methods.

### Verify CSS Prefixing in Views
Check that ALL generated Blade views use the module's Tailwind prefix (`{prefix}:utility`).
If not, fix them — the `module:make-crud` stubs may not include prefixes.

### Update Menu Service
Verify the new resource was added to the module's MenuService with correct permissions.

6. Run `php artisan migrate` to apply the permission migration
7. Run `vendor/bin/pint --dirty` to fix code style
8. Show a summary of all generated files and available routes

## Field Types
- Basic: `string`, `text`, `integer`, `boolean`, `date`, `datetime`, `decimal`, `json`
- UI-specific: `toggle` (switch), `select:Option1|Option2` (dropdown), `editor` (rich text), `media` (media library), `file`
- Layout: `--columns=2` for two-column form layout

## Examples
```
/new-crud Crm Contact name:string,email:string,phone:string,type:select:Lead|Customer|Partner,notes:text
/new-crud Blog Post title:string,content:editor,status:select:Draft|Published,featured_image:media
/new-crud Crm --migration=create_crm_deals_table
```

## Post-Generation Checklist
- [ ] Model fillable/casts correct
- [ ] Routes registered: `php artisan route:list --name={module}`
- [ ] Menu item added to MenuService
- [ ] Hook enums created and wired into service
- [ ] CSS classes prefixed in all generated views
- [ ] Policy permissions match expected names
- [ ] Tests run: `php artisan test --filter={Model}`
